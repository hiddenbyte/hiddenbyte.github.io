There are many languages and/or runtimes that enables to start two or more threads of exectution, Go enables it by introducing the concept of goroutine. A goroutine is a thread of execution managed by the Go runtime.

Before digging into goroutines let's understand the difference between calling a function and launching a goroutine. The code bellow shows two function declarations, one named main - the program entry point - and other named sleepyReceiver called by the main function. 

  func main() {
    sleepyReceiver("some message")
  }

  func sleepyReceiver(message string) {
    time.Sleep(5 * time.Second)
    log.Printf("sleepy: Received message '%v'", message)
  }

If we compile this code - go build main.go - and then disassemble the resulting executable using objdump - go tool objdump main - we can see that the sleepyReceiver("some message") function call was compiled into a stream of Assembly instructions bellow. 

  LEAQ go.string.*+4341(SB), AX
  MOVQ AX, 0(SP)
  MOVQ $0xc, 0x8(SP)
  CALL main.sleepyReceiver(SB)

Nothing out of ordinary here, the "some message" argument is being copied to stack and then the sleepyReceiver function is being called.

Launching a goroutine is just as simple as calling a function and can be done using go statements.

  func main() {
    go sleepyReceiver("some message")
  }
  

  MOVL $0x10, 0(SP)
  LEAQ go.func.*+121(SB), AX
  MOVQ AX, 0x8(SP)
  LEAQ go.string.*+4341(SB), AX
  MOVQ AX, 0x10(SP)
  MOVQ $0xc, 0x18(SP)
  CALL runtime.newproc(SB)
