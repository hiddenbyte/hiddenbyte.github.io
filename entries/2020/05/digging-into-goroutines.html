Digging into Goroutines - Launching

There are many languages and/or runtimes that enables to start two or more threads of exectution, Go enables it by introducing the concept of goroutine. A goroutine is a thread of execution managed by the Go runtime.

Before digging into goroutines let's understand the difference between calling a function and launching a goroutine.

Calling a function

The code bellow shows two function declarations, one named main - the program entry point - and other named sleepyReceiver called by the main function.

func main() { sleepyReceiver("some message") } func sleepyReceiver(message string) { time.Sleep(2 * time.Second) log.Printf("sleepy: Received message '%v'", message) }

If we compile this code - go build main.go - and then disassemble the resulting executable using objdump - go tool objdump main - we can see that the sleepyReceiver("some message") function call was compiled into a stream of Assembly instructions bellow.

LEAQ go.string.*+4341(SB), AX MOVQ AX, 0(SP) MOVQ $0xc, 0x8(SP) CALL main.sleepyReceiver(SB)

Nothing out of the ordinary here, the "some message" argument is being copied to the stack and then the sleepyReceiver function is being called.

Launching a goroutine

Launching a goroutine is just as simple as calling a function and can be achieved by writing "go" statements.

A "go" statement starts the execution of a function call as an independent concurrent thread of control, or goroutine, within the same address space.

in The Go Programming Language Specification

Now we are going to make a slight change in the "Calling a function" example, we are going to call sleepyReceiver but start the execution of the function call in a different thread, by launching a goroutine, this can be achieved by adding the go keyword before the function call, as you can see bellow.

func main() { go sleepyReceiver("some message") }

An important aspect to notice here: we didn't make any change on sleepyReceiver in order launch as a goroutine.

MOVL $0x10, 0(SP) LEAQ go.func.*+121(SB), AX MOVQ AX, 0x8(SP) LEAQ go.string.*+4341(SB), AX MOVQ AX, 0x10(SP) MOVQ $0xc, 0x18(SP) CALL runtime.newproc(SB)
